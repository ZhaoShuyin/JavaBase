项目中问题反馈的审批流程
1.
1.1 work_code流程的编号，isstart是否启用，create_user创建人,create_time创建时间，
1.2 ID工作流的标识，
    work_name流程节点的名称，assign_id受理人，
    handle_data处理日期，remakes处理结果（备注），audit_status审核状态，
    is_audit审核标识，document_code申请单据号，flow_number流程序号
2. 审批系统的登录注册，根据不同的ID信息信息不同的内容，问题具体内容的编辑、问题是否确实存在的审批、问题需要解决的分配、问题解决之后的反馈。
3.问题内容的编辑，编写人的姓名，问题的编号问题的内容，提交问题的时间
4.配置审核流程，
ID	flow_number	Flow_name
xx	1	产品经理
xx	2	开发主管
xx	3	开发人员
5.问题内容编辑完成之后点击提交，提交之后后台改变状态字段为送审状态，
  然后产品经理登陆系统，根据产品经理ID对应显示产品经理需要审核所有申请，产品经理点击审核，
  根据申请单据号显示需要解决的反馈问题的详情，选择通过或者驳回，根据需要填写反馈意见，
  然后选择提交，后台对应改变审核状态的状态码，
6.送审问题
6.1如果是最后一步
通过{
	编写处理通过的业务代码，开发人员更改问题处理状态
}若不通过{
	直接改变审核状态后台字段为驳回状态
}
6.2如果不是最后一步
通过{
	取出流程序号+1，进入下一步流程操作
}若不通过{
	直接改变审核状态后台字段为驳回状态
}
